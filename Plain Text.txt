## EN: Release Notes — Version 1.3.0
### Key Improvements:
* **Performance & Architecture:** Refactored `content.js` into a robust `SellersInspector` class modular structure. Implemented **virtual scrolling** (line-window rendering and spacers) to prevent DOM freezing when processing massive JSON files. Moved regex-heavy syntax highlighting and badge injection entirely off the main thread via asynchronous message passing to the background service worker.
* **New Analytics & Stats:** Added specific counters for seller types (`PUBLISHER`, `INTERMEDIARY`, `BOTH`) and `is_confidential` records in the overview panel. Introduced an actionable link icon next to `Ads: OK` status to directly open `https://<domain>/ads.txt`.
* **Enhanced UI & Filtering:** Introduced a local overview search box to instantly filter rendered JSON lines by `seller_id`, `domain`, or `name`. Added a complete **Light Theme** toggle in the popup menu, powered by localized CSS variables.
* **UI/UX Viewport Refactoring:** Redesigned the `.json-virtual-viewport` layout to `100vh`, seamlessly integrating the scrollbar with the browser edge and eliminating the "double scrollbar" artifact.
* **Export & Modal Upgrades:** The filtered data modal now supports A-Z / Z-A sorting for Domain and Name lists, alongside robust `Save .csv` functionality with proper row value escaping.
* **Configuration & Wiring:** Introduced a new `shared-config.js` module to unify default settings and initialization order across the popup and content scripts, drastically improving maintainability.

### Bug Fixes:
* **CSP SecurityError Resolution:** Fixed a critical crash on domains enforcing strict Content Security Policies (such as adWMG) by completely abandoning external Web Workers. Highlighting logic is now executed securely within the background script. Removed `web_accessible_resources` to harden extension security.
* **Runtime Reliability:** Fixed structural vulnerabilities by introducing robust null-guards, safer async progress handling, centralized concurrency execution, and resilient filtering logic for edge cases (missing arrays, missing IDs, invalid domains, and empty inputs).
* **Popup Tab Handling:** Reworked popup query logic to prevent execution failures and unexpected behavior when activated on restricted browser tabs or invalid active tab states.
